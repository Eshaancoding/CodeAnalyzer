Visualizer Tutorial
Welcome to this tutorial! We are going to take a look at how this website works. This tutorial is going to take a look at 2 categories:


- Visualizing your code, using the top dark-grey area
- Editing your code, using the bottom dark-grey area

Take a look at the samples if you feel confused. If you're code seems right, yet the program is compiled successfully, please email me at eshaanbarkataki@gmail.com.

Let's go!
Editing:
The compiler accepts Javascript code. There is also syntax highlighting enabled. However, there are a few conditions to successfully compile:

- No function declared that is named visualize. That's the function that (hence the name) displays the variables. You don't want to replace it.

- All variables are declared without var or let. For example, a valid variable declaration would be a = 5 but not var a = 5 or let a = 5.

- Don't use global variables! You may mess around with the variables in the source code, which will affect the website. Declare your variables with let or var

Running:
Here's what you should keep in mind when running your program:

- You could visualize your code by adding visualize() onto a seperate line.

- Hit the green button on the bottom that says "Run", and it will run the code for you.

- There is a delay every time visualize() is called. This delay is adjustable using the text area on the bottom left. Just enter your delay in milliseconds.

- When you execute your code, you cannot stop it. However, if your code is taking too long, just set the delay to 0 ms.

- If there are any errors in your code, then the editor will notify you. The error notification will hide once you move your cursor or type anything.

- the visualize functions is not allowed inside other functions.
